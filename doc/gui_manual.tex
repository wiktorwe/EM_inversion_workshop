This manual describes how to set up a Python environment, configure machine-specific paths, install dependencies, and run the seven workshop GUIs. For each GUI it explains how to launch it, how to set parameters, and—for non-expert users—what each parameter means physically and what impact it has on the simulation or workflow.

The seven GUIs are:
\begin{itemize}
  \item \textbf{00\_configure\_workshop} — Set rockem-suite path, MPI launcher, default SEG-Y, optional GPU engines for 2D forward/inversion, and other machine-specific settings.
  \item \textbf{01\_fw\_setup} — Prepare FD modelling inputs from a SEG-Y file.
  \item \textbf{02\_fwmodelling\_and\_data\_visualization} — Run FD modelling and visualize channel-gain amplitude/phase from shot records.
  \item \textbf{03\_2d\_inversion} — Generate inversion inputs and run the 2D inversion locally.
  \item \textbf{04\_2d\_inversion\_results} — View 2D inversion results and compare real data with synthetics.
  \item \textbf{05\_1d\_inversion} — Independent 1D layered inversion, per transmitter, against a validated analytic magnetic (Hx) line-source solver.
  \item \textbf{06\_1d\_inversion\_results} — View the 1D inversion's pseudo-2D section, per-Tx diagnostics, and modelled-vs-observed data comparison.
\end{itemize}

Steps 00--04 form the 2D explicit-FDTD forward/inversion workflow; 05--06 are an independent, complementary 1D analysis that runs directly against the 2D forward run's own data (no separate FDTD run needed) and does not depend on 03/04's inversion having been run.

All GUIs are Jupyter notebooks served as code-hidden applications via Voila (ipywidgets and Plotly). Run Voila from the \textbf{workshop root} so that \texttt{scripts} and \texttt{third\_party} are on the Python path.

\section{Workspace layout}
%=============================================================================

All notebook-generated artifacts are written under \texttt{workspace/} (gitignored). This keeps the repository root clean and separates 2D and 1D workflows:

\begin{verbatim}
workspace/
  2D/
    forward/              # Step 01-02: FD model, mod.cfg, shot gathers
    inversion/
      input/              # Step 03: prepared inversion inputs
      Run{N}/             # Step 03: 2D inversion runs (Run0, Run1, ...)
    results/Run{N}/       # Step 04: SEG-Y exports
  1D/
    inversion/Run{N}/     # Step 05: 1D inversion runs
    results/Run{N}/       # Step 06: SEG-Y exports
\end{verbatim}

\textbf{Example model:} \texttt{examples/Fault\_1.sgy} is a sample resistivity model committed with the repository. Load it in Step 01 (or set it as the default in Step 00).

\textbf{Restore pristine state:} run \texttt{./clean.sh} from the workshop root to remove \texttt{workspace/}, Voila PID files, and caches. Use \texttt{./clean.sh --dry-run} to preview, or \texttt{./clean.sh -y} to skip confirmation.

\section{Environment setup}
%=============================================================================

\subsection{System prerequisites}

Before setting up Python, ensure you have:
\begin{itemize}
  \item Python 3.10 or newer
  \item \texttt{mpirun} (OpenMPI or compatible MPI runtime)
  \item A validated \texttt{rockem-suite} checkout, built with \texttt{make mpi}, with the \textbf{explicit} TE2D engine binaries \texttt{mpiEmmodTE2d} and \texttt{mpiEminvTE2d} under its \texttt{bin/} directory. Their location is \texttt{\$ROCKEM\_SUITE\_ROOT/bin}, where \texttt{ROCKEM\_SUITE\_ROOT} is set by Step 00 (or defaults to \texttt{\textasciitilde/software/new\_rockem/rockem-suite}; see \texttt{scripts/modules/rockem\_bridge.py}). The ADI TE2D engine (\texttt{mpiEmmodADITE2d}/\texttt{mpiEminvADITE2d}) is \textbf{not} used by this workshop — it fails rockem-suite's own layered-model Green's-function validation.
\end{itemize}

\textbf{Optional — GPU acceleration (2D TE2D only):} On a machine with an NVIDIA GPU and CUDA toolkit, you can build the GPU variants \texttt{mpiEmmodTE2dGpu} and \texttt{mpiEminvTE2dGpu} in rockem-suite:
\begin{quote}
  \texttt{make -f Makefile.gpu MPI\_CXX=mpic++ RS\_CUDA\_ARCH=<arch> CUDA\_HOME=\$CUDA\_HOME \textbackslash}\\
  \texttt{\ \ \ \ bin/mpiEmmodTE2dGpu bin/mpiEminvTE2dGpu}
\end{quote}
Enable them in Step 00 (\textbf{Use GPU for 2D forward/inversion}). Step 00 \textbf{Validate} checks \texttt{nvidia-smi} and the GPU binaries when those options are on. GPU forward requires re-running Step 01 \textbf{Finalize setup} so \texttt{setup\_metadata.json} records the GPU engine; GPU inversion is applied when Step 03 writes \texttt{runinv.sh}.

\subsection{Creating a virtual environment (venv)}

From the workshop root:
\begin{quote}
  \texttt{python3 -m venv .venv}\\
  \texttt{source .venv/bin/activate} \quad (Linux/macOS)\\
  \texttt{.venv\textbackslash Scripts\textbackslash activate} \quad (Windows)
\end{quote}
Then upgrade pip: \texttt{pip install --upgrade pip}.

\subsection{Creating a conda environment}

\begin{quote}
  \texttt{conda create -n em\_workshop python=3.10}\\
  \texttt{conda activate em\_workshop}
\end{quote}

\subsection{Installing dependencies}

With the virtual or conda environment activated, install:

\begin{center}
\begin{tabular}{ll}
  \toprule
  Package    & Purpose \\
  \midrule
  \texttt{voila}      & Serves notebooks as standalone web apps (code hidden). \\
  \texttt{ipywidgets}& Interactive widgets in the GUI. \\
  \texttt{plotly}    & Interactive plots. \\
  \texttt{numpy}     & Numerical arrays (used by inversion and results GUIs). \\
  \texttt{ipykernel} & Jupyter kernel for Python (required for Voila to run notebooks). \\
  \texttt{matplotlib}& Plotting (used by workshop modules). \\
  \texttt{scipy}     & Scientific computing (used by workshop modules). \\
  \texttt{segyio}    & Read/write SEG-Y files (required by 01\_fw\_setup). \\
  \texttt{joblib}    & Parallelizes 05's Tx $\times$ seed ensemble inversion across CPU cores. \\
  \bottomrule
\end{tabular}
\end{center}

Command:
\begin{quote}
  \texttt{pip install voila ipywidgets plotly numpy ipykernel matplotlib scipy segyio joblib}
\end{quote}

\textbf{Conda users:} After installing, register the kernel so Voila can find it:
\begin{quote}
  \texttt{python -m ipykernel install --user --name=em\_workshop --display-name="Python (em\_workshop)"}
\end{quote}

\section{How to run the GUIs}
%=============================================================================

From the \textbf{workshop root} directory, use the provided shell scripts. Each script changes into the script directory and starts Voila with the corresponding notebook (sources stripped so users see only the interface).

\begin{itemize}
  \item \texttt{./start\_00\_configure.sh} \quad $\to$ 00\_configure\_workshop
  \item \texttt{./start\_01\_fw\_setup.sh} \quad $\to$ 01\_fw\_setup
  \item \texttt{./start\_02\_fwmodelling\_and\_data\_visualization.sh} \quad $\to$ 02\_fwmodelling\_and\_data\_visualization
  \item \texttt{./start\_03\_2d\_inversion.sh} \quad $\to$ 03\_2d\_inversion
  \item \texttt{./start\_04\_2d\_inversion\_results.sh} \quad $\to$ 04\_2d\_inversion\_results
  \item \texttt{./start\_05\_1d\_inversion.sh} \quad $\to$ 05\_1d\_inversion
  \item \texttt{./start\_06\_1d\_inversion\_results.sh} \quad $\to$ 06\_1d\_inversion\_results
\end{itemize}

\textbf{Recommended order:} run Step 00 once per machine, then 01 $\to$ 02 $\to$ 03 $\to$ 04 for the 2D workflow. Steps 05--06 can follow after 01/02 have produced FD data for at least one transmitter — they read directly from that data and do not require 03/04's 2D inversion to have been run.

Alternatively, from the workshop root:
\begin{quote}
  \texttt{voila 00\_configure\_workshop.ipynb --strip\_sources=True}
\end{quote}
(and similarly for the other notebooks.)

Each start script writes its own Voila PID file (e.g.\ \texttt{.voila\_fw\_setup\_server.pid}, \texttt{.voila\_2d\_inversion\_server.pid}), so multiple GUIs can be managed independently.

\section{Recommended workflow}
%=============================================================================

\begin{enumerate}
  \item \textbf{Step 00 — Configure} — Set \texttt{rockem-suite} root, MPI launcher (\texttt{mpirun}), default process count, default SEG-Y (\texttt{examples/Fault\_1.sgy}), workspace directory name, and optional GPU flags for 2D forward (\texttt{mpiEmmodTE2dGpu}) and inversion (\texttt{mpiEminvTE2dGpu}). Click \textbf{Validate} to check CPU/GPU binaries, \texttt{nvidia-smi}, and paths, then \textbf{Save} to write \texttt{workshop\_config.json}.
  \item \textbf{Step 01 — FW setup} — Load a SEG-Y resistivity model, configure survey and FD design, then \textbf{Generate FD inputs (Finalize setup)}. Outputs go to \texttt{workspace/2D/forward/}.
  \item \textbf{Step 02 — FW modelling and data visualization} — Run forward modelling, load \texttt{Hxshot.rss}/\texttt{Hzshot.rss}, compute channel-gain amplitude/phase, and inspect results.
  \item \textbf{Step 03 — 2D inversion} — Generate inversion inputs into \texttt{workspace/2D/inversion/input/}, then run inversion into \texttt{workspace/2D/inversion/RunN/}.
  \item \textbf{Step 04 — 2D inversion results} — Compare models and data; export SEG-Y to \texttt{workspace/2D/results/RunN/}.
  \item \textbf{Step 05 — 1D inversion} — Run per-transmitter 1D layered inversion; outputs go to \texttt{workspace/1D/inversion/RunN/}.
  \item \textbf{Step 06 — 1D inversion results} — View pseudo-2D sections and export to \texttt{workspace/1D/results/RunN/}.
\end{enumerate}

\section{GUI 0: 00\_configure\_workshop}
%=============================================================================

\subsection{Purpose}

Configure machine-specific paths before running the other notebooks. Settings are saved to \texttt{workshop\_config.json} at the repo root (gitignored). All other notebooks read this file at startup via \texttt{scripts/modules/workshop\_config.py}.

\subsection{Parameters}

\begin{itemize}
  \item \textbf{rockem-suite} — Path to the validated rockem-suite checkout (must contain \texttt{python/}, \texttt{bin/mpiEmmodTE2d}, \texttt{bin/mpiEminvTE2d}, and the analytic Green's-function solver under \texttt{doc/examples/validate\_layered\_1d\_model/shared/}).
  \item \textbf{mpirun} — MPI launcher executable (usually \texttt{mpirun}). Used by Steps 02, 03, and 04 when launching FD or inversion jobs.
  \item \textbf{nproc} — Default number of MPI processes for FD and inversion runs (can still be changed per GUI).
  \item \textbf{default SEG-Y} — Default resistivity model path pre-filled in Step 01 and used as fallback for SEG-Y export in Steps 04 and 06. The workshop includes \texttt{examples/Fault\_1.sgy}.
  \item \textbf{workspace} — Name of the generated-artifacts directory (default \texttt{workspace}).
  \item \textbf{Use GPU for 2D forward} — When enabled, Steps 01, 02, and 04 use \texttt{mpiEmmodTE2dGpu} instead of \texttt{mpiEmmodTE2d}. Requires the GPU binary and a working NVIDIA driver (\texttt{nvidia-smi}). After enabling, re-run Step 01 \textbf{Finalize setup}.
  \item \textbf{Use GPU for 2D inversion} — When enabled, Step 03 writes \texttt{runinv.sh} with \texttt{mpiEminvTE2dGpu} instead of \texttt{mpiEminvTE2d}.
  \item \textbf{Buttons} — \textbf{Validate} checks CPU binaries (required), GPU binaries and \texttt{nvidia-smi} (required when GPU options are on; informational otherwise), \texttt{mpirun}, default SEG-Y, and the analytic solver; \textbf{Save} writes \texttt{workshop\_config.json}; \textbf{Reset defaults} restores built-in defaults without saving; \textbf{Quit Voila} stops the server.
\end{itemize}

\subsection{Configuration file fields}

When saved, \texttt{workshop\_config.json} includes:
\begin{itemize}
  \item \texttt{rockem\_suite\_root}, \texttt{mpirun}, \texttt{nproc\_default}, \texttt{default\_segy}, \texttt{workspace\_dir}
  \item \texttt{use\_gpu\_forward\_2d} — \texttt{true}/\texttt{false}
  \item \texttt{use\_gpu\_inversion\_2d} — \texttt{true}/\texttt{false}
\end{itemize}

\section{GUI 1: 01\_fw\_setup}
%=============================================================================

\subsection{Purpose}

Generate finite-difference (FD) modelling inputs from a SEG-Y resistivity model: conductivity/permittivity files (\texttt{sg.rss}, \texttt{ep.rss}), source wavelet (\texttt{wav2d.rss}), survey geometry (\texttt{Survey.rss}), and \texttt{mod.cfg} settings. Intermediate files are written to a temporary session folder; final outputs go to \texttt{workspace/2D/forward/}.

\subsection{Parameters (physical meaning and impact)}

\begin{itemize}

  \item \textbf{SEGY file} — Path to the input SEG-Y file containing the resistivity model. Defaults to \texttt{examples/Fault\_1.sgy} if configured in Step 00. Uniform permittivity for \texttt{ep.rss} is set from the FD design (\texttt{eps\_r\_used} in section 4 / \texttt{setup\_metadata.json}), not from a separate constant.

  \item \textbf{tx0 (m), tz0 (m), dtx (m), ntx} — Transmitter (source) geometry: first source position in $x$ and $z$, spacing between sources, and number of sources. All positions must lie inside the model bounds.

  \item \textbf{rx0 (m), rz0 (m), drx (m), nrx} — Receiver geometry: first receiver $x$ and $z$, receiver spacing, and number of receivers.

  \item \textbf{flist (Hz)} — Comma-separated list of source frequencies (e.g.\ \texttt{2000,4000,6000}). Lower frequencies penetrate deeper; higher frequencies resolve shallow structure better.

  \item \textbf{dt (s)} — Time step for the wavelet and for the FD solver. It must satisfy numerical stability (CFL) and sample the highest frequency.

  \item \textbf{n\_periods} — Number of periods of the lowest frequency in the wavelet. Controls how long the source radiates.

  \item \textbf{alpha} — Ramp parameter (0--1) for the ramp square-wave wavelet.

  \item \textbf{f\_min (Hz), f\_max (Hz)} — Frequency range used to design the FD grid for the \textbf{explicit} engine (\texttt{mpiEmmodTE2d} or \texttt{mpiEmmodTE2dGpu} when GPU forward is enabled in Step 00). Grid spacing, time step, \texttt{eps\_r}, PML, and depth margin follow deterministically from these plus \texttt{rho\_min}/\texttt{rho\_max} and receiver offsets.

  \item \textbf{rho\_min (Ohm-m), rho\_max (Ohm-m)} — Expected resistivity range in the model. Grid spacing is sized from skin depth and minimum receiver offset; \texttt{eps\_r} is sized from \texttt{rho\_max}/\texttt{f\_max}.

  \item \textbf{dimension} — 2D or 3D. \textbf{Note:} the 3D path uses the legacy ADI engine and is not covered by this workshop's explicit-engine validation.

  \item \textbf{points/skin, cells/min offset, tan(delta) floor, eps\_r cap, sigma\_max clip} — FD design controls; see the GUI readout after \textbf{Compute FD design}.

  \item \textbf{Depth margin (auto)} — If the loaded model is too shallow for source/receiver clearance, \textbf{Generate FD inputs} automatically pads it in depth.

  \item \textbf{Buttons} — \textbf{Load model}, \textbf{Plot model}, \textbf{Build intermediate sg/ep}, \textbf{Build intermediate wav2d}, \textbf{Generate survey.cfg + Survey.rss}, \textbf{Compute FD design}, \textbf{Generate FD inputs (Finalize setup)} writes everything to \texttt{workspace/2D/forward/}. Use \textbf{Quit GUI server} to stop Voila.

\end{itemize}

%=============================================================================
\section{GUI 2: 02\_fwmodelling\_and\_data\_visualization}
%=============================================================================

\subsection{Purpose}

Run FD modelling (via \texttt{workspace/2D/forward/runmod.sh} and the explicit-engine MPI binary selected in Step 00), load \texttt{Hxshot.rss} and \texttt{Hzshot.rss} outputs from \texttt{workspace/2D/forward/Data/}, compute steady-state channel-gain amplitude and phase, and visualize or save the results.

\subsection{External requirement}

The explicit-engine MPI FD binary must be installed: \texttt{mpiEmmodTE2d} (CPU) or \texttt{mpiEmmodTE2dGpu} (GPU). The engine name is stored in \texttt{workspace/2D/forward/setup\_metadata.json} when Step 01 finalizes setup. Step 00 (or \texttt{ROCKEM\_SUITE\_ROOT}) must point at a valid checkout; \texttt{runmod.sh} reads the engine from that metadata.

\subsection{Parameters (physical meaning and impact)}

\begin{itemize}

  \item \textbf{nproc} — Number of MPI processes for the FD run (default from Step 00 config).

  \item \textbf{f\_min (Hz), n\_periods (of f\_min)} — Reference frequency and extraction window for steady-state phasor extraction (same convention as Step 05).

  \item \textbf{component} — Hx or Hz magnetic field component.

  \item \textbf{plot} (metric) — Amplitude or phase vs receiver, transmitter, or frequency.

  \item \textbf{frequency, tx, local rx} — Selectors for display.

  \item \textbf{Buttons} — \textbf{Run modelling (background)}, \textbf{Stop run}, \textbf{Refresh run status}, \textbf{Load FD outputs}, \textbf{Compute channel-gain amplitude/phase}, \textbf{Save processed results} (writes NPZ under \texttt{Data/processed/}), \textbf{Quit GUI server}.

\end{itemize}

\section{GUI 3: 03\_2d\_inversion}
%=============================================================================

\subsection{Purpose}

Generate inversion inputs from \texttt{workspace/2D/forward/} and the inversion template, run the inversion locally (staged into the next \texttt{workspace/2D/inversion/RunN/} directory), and monitor logs and model progress.

\subsection{External requirement}

The explicit-engine MPI inversion binary must be installed: \texttt{mpiEminvTE2d} (CPU) or \texttt{mpiEminvTE2dGpu} (GPU). Step 03 writes \texttt{runinv.sh} with the engine selected in Step 00 (\texttt{scripts/modules/workshop\_config.patch\_runinv\_template}).

\subsection{Parameters (physical meaning and impact)}

\begin{itemize}

  \item \textbf{Initial model} — Uniform conductivity (sigma) or resistivity (rho) starting model.

  \item \textbf{sigma (S/m), rho (Ohm.m)} — Value of the uniform initial model.

  \item \textbf{Max iter} — Maximum inversion iterations.

  \item \textbf{apertx (m), dtx, dtz} — Grid and extent parameters. \texttt{apertx} defaults to the forward run's value from Step 01 when available.

  \item \textbf{Tikhonov (sg)} — Regularization weight (\texttt{tik\_sgregalpha}).

  \item \textbf{Clean selected RunN before run} — Clear the chosen run directory before staging new inputs.

  \item \textbf{Load run} — Select which \texttt{RunN} under \texttt{workspace/2D/inversion/} to monitor.

  \item \textbf{nproc} — MPI processes for inversion (default from Step 00).

  \item \textbf{Buttons} — \textbf{Generate inversion inputs} writes to \texttt{workspace/2D/inversion/input/}; \textbf{Run inversion locally} stages into \texttt{workspace/2D/inversion/RunN/}; \textbf{Refresh from progress.log}; \textbf{Stop run}; \textbf{Quit GUI server}.

\end{itemize}

%=============================================================================
\section{GUI 4: 04\_2d\_inversion\_results}
%=============================================================================

\subsection{Purpose}

View 2D inversion results: select a run (\texttt{RunN}) and model (\texttt{sg0}, \texttt{sg\_ls} at an iteration, etc.), view 2D resistivity with true-model comparison, resistivity slices, compare real data with synthetics, and export selected models to SEG-Y.

\subsection{Parameters (physical meaning and impact)}

\begin{itemize}

  \item \textbf{Run, Model} — Which inversion run under \texttt{workspace/2D/inversion/} and which model file.

  \item \textbf{Export selected model to SEGY} — Writes to \texttt{workspace/2D/results/RunN/} using the setup SEG-Y as a template. Interpolates to template geometry if needed.

  \item \textbf{Export naming} — e.g.\ \texttt{sg\_up\_iter011.segy}, \texttt{sg0\_iter000.segy}, \texttt{sg\_ls\_iterNNN.segy}.

  \item \textbf{x (m), z (m), Freqs (Hz)} — Slice positions and frequencies for comparison.

  \item \textbf{Load real data / Load synthetics from run / Generate synthetics from model} — Data comparison options. \textbf{Generate synthetics} runs FD for the selected model using \texttt{nproc} MPI processes. It always refreshes \texttt{ep.rss} from \texttt{workspace/2D/forward/} and checks that its permittivity matches \texttt{eps\_r\_used} in \texttt{setup\_metadata.json} from Step 01 (not the legacy ADI default of 7).

  \item \textbf{component, plot, frequency, tx, local rx, trace idx} — Display selectors (same as Step 02).

  \item \textbf{Quit GUI server} — Stops Voila.

\end{itemize}

%=============================================================================
\section{GUI 5: 05\_1d\_inversion}
%=============================================================================

\subsection{Purpose}

An independent 1D layered inversion, run separately for each transmitter, against rockem-suite's validated analytic magnetic (Hx) line-source Green's function. Reads channel-gain data from \texttt{workspace/2D/forward/Data/} (same phasor extraction as Step 02) — it does not need Steps 03/04 to have been run. Each transmitter is calibrated against the analytic solver on the blocky 1D model beneath it, then inverted independently; results are combined into a pseudo-2D section under \texttt{workspace/1D/inversion/RunN/}.

\subsection{Parameters (physical meaning and impact)}

\begin{itemize}

  \item \textbf{Freqs (Hz), n\_periods (of f\_min)} — Channel-gain extraction settings (same convention as Step 02).

  \item \textbf{n\_layers, z\_start, z\_end, rho\_bg, rho\_min, rho\_max, thk\_min, thk\_max} — 1D model parameterization and search bounds.

  \item \textbf{optimizer} — Differential Evolution, Dual Annealing, or BlockInv (Gauss--Newton).

  \item \textbf{calibration} — Fixed calibration constant (recommended) or per-frequency source factor.

  \item \textbf{maxiter, popsize, maxfun, seed, N runs/Tx, n\_jobs, blk\_iter} — Optimizer and ensemble settings.

  \item \textbf{w\_hxh, w\_hxhz, Tikhonov lambda} — Misfit weights and smoothness penalty.

  \item \textbf{dx\_out, dz\_out, zmax (0=auto)} — Pseudo-2D output grid.

  \item \textbf{Buttons} — \textbf{Reload features}, \textbf{Check kx convergence}, \textbf{Run all Tx inversions}, \textbf{Build pseudo-2D section}, \textbf{Export results} (NPZ/JSON under current \texttt{RunN}), \textbf{Refresh QC plots}, \textbf{Quit GUI server}.

\end{itemize}

\section{Troubleshooting}
%=============================================================================

\begin{itemize}
  \item \textbf{Missing configuration} — Run Step 00 and click \textbf{Save} so \texttt{workshop\_config.json} exists. Without it, defaults apply (\texttt{\textasciitilde/software/new\_rockem/rockem-suite}, \texttt{examples/Fault\_1.sgy}).
  \item \textbf{Missing \texttt{segyio}} — Install \texttt{segyio} and start Voila from the workshop root.
  \item \textbf{No Jupyter kernel for language `python' found} — Install \texttt{ipykernel} and register the \texttt{em\_workshop} kernel (see Environment setup).
  \item \textbf{SEGY export cannot find template} — Export expects \texttt{segy\_template\_path} in \texttt{workspace/2D/forward/setup\_metadata.json}, with fallback to the default SEG-Y from Step 00. Run Step 01 and finalize setup first.
  \item \textbf{Missing explicit-engine MPI FD binary} — Steps 02 and 04 need \texttt{mpiEmmodTE2d} (or \texttt{mpiEmmodTE2dGpu} if GPU forward is enabled). Validate in Step 00; build CPU with \texttt{make mpi}, GPU with \texttt{make -f Makefile.gpu bin/mpiEmmodTE2dGpu}.
  \item \textbf{Missing explicit-engine MPI inversion binary} — Step 03 needs \texttt{mpiEminvTE2d} (or \texttt{mpiEminvTE2dGpu} if GPU inversion is enabled).
  \item \textbf{GPU validation fails} — Ensure \texttt{nvidia-smi} works, CUDA is installed, and GPU binaries exist under \texttt{\$ROCKEM\_SUITE\_ROOT/bin/}. Build targets in rockem-suite as shown in Environment setup. If you enabled GPU forward after Step 01, re-run \textbf{Finalize setup} in Step 01.
  \item \textbf{Wrong/stale rockem-suite checkout} — Step 00 validation checks for \texttt{magnetic\_line\_source\_fields\_layered} in the analytic solver shared directory.
  \item \textbf{Missing \texttt{joblib}} — Step 05 needs \texttt{joblib} for parallel ensemble runs.
  \item \textbf{Stale artifacts} — Run \texttt{./clean.sh} to remove \texttt{workspace/} and start fresh.
\end{itemize}

\section{File locations}
%=============================================================================

\begin{itemize}
  \item \texttt{workshop\_config.json} — Machine-specific settings (Step 00; gitignored): rockem-suite path, MPI launcher, default SEG-Y, workspace dir, \texttt{use\_gpu\_forward\_2d}, \texttt{use\_gpu\_inversion\_2d}.
  \item \texttt{workspace/2D/forward/} — FD inputs and outputs (\texttt{mod.cfg}, \texttt{sg.rss}, \texttt{Survey.rss}, \texttt{Data/}).
  \item \texttt{workspace/2D/inversion/input/} — 2D inversion input files (\texttt{inv.cfg}, \texttt{sg0.rss}, etc.).
  \item \texttt{workspace/2D/inversion/Run0/}, \texttt{Run1/}, \ldots — 2D inversion run directories (Steps 03/04).
  \item \texttt{workspace/2D/results/RunN/} — SEG-Y exports from Step 04.
  \item \texttt{workspace/1D/inversion/Run0/}, \ldots — 1D inversion run directories (Steps 05/06).
